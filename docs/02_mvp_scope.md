# 02. MVP 기능 범위

## 접근 권한

| 구분 | 가능한 기능 |
| --- | --- |
| **비로그인** | F-1 전체 (지도 조회, 필터, 실거래가 레이어, 안전 레이어), F-4 시세·안전 분석 조회 |
| **로그인** | F-1 + F-2~F-8 전체 (AI 에이전트, 법률 상담, 자연어 기반 분석 요청, HUG 계산, 찜하기, 대화 세션·요약 메모리) |

## 개발 단계

| 단계 | 포함 기능 |
| --- | --- |
| **MVP** | F-1 전체, F-2 매물 추천, F-3 법률 상담 RAG, F-4 시세·안전 분석, F-6 기본 로그인, F-7 찜하기 |
| **1.5차** | F-4 뉴스 RAG, F-5 HUG 간이 계산, F-6 소셜 로그인, F-8 대화 세션·요약 메모리 |
| **확장** | 실제 매물 크롤링/제휴, 정교한 HUG/HF/SGI 판정, 등기부등본 AI 분석 |

> MVP 핵심: 지도 기반 부동산 탐색 (전월세·매매), AI 에이전트 + 지도 연동, 법률 RAG, 시세·안전 분석
> 뉴스 RAG, HUG 정밀 판정, 소셜 로그인, 대화 기록은 MVP 이후 단계

---

## F-1. 지도 기반 매물 탐색

> 메인 화면 — 비로그인 포함 전체 공개

| 기능 | 상세 설명 | 데이터 소스 | 단계 |
| --- | --- | --- | --- |
| **매물 마커 표시** | 지도 진입 시 전체 매물이 마커로 표시됨. 마커 클릭 시 매물명, 보증금/월세, 평수, 층수, 등록일 팝업 노출. 줌 레벨에 따라 클러스터링 처리. | 더미 매물 DB (MVP) → 네이버부동산 크롤링 또는 제휴 API (확장) | MVP |
| **필터** | 동/구 선택(드롭다운), 거래유형(전세/월세/매매), 매물유형(원룸·오피스텔·아파트·연립다세대·단독다가구), 가격 범위 슬라이더. 필터 적용 시 지도 마커 즉시 업데이트. | PostgreSQL 쿼리 필터링. 네이버지도 API 행정동 코드 연계. | MVP |
| **실거래가 레이어** | 지도 우상단 레이어 토글. 매물 마커 클릭 시 동일 면적 평균 대비 현재 매물 가격 비교 팝업. 아파트/오피스텔/연립다세대/단독다가구 유형별 분리. | 국토교통부 전월세·매매 실거래가 공공 API 8종 → 주기적 수집 후 DB 저장 | MVP |
| **안전 레이어** | 지도 우상단 레이어 토글. 비상벨 위치, 범죄주의구간, 보안등, CCTV, 치안안전시설(파출소) 오버레이. 복수 레이어 동시 활성화 가능. | 생활안전지도 공공 API 5종 → 좌표 데이터 → 네이버지도 오버레이 | MVP |

## F-2. AI 에이전트 — 매물 추천

> 챗봇 기반 / 로그인 필요

| 기능 | 상세 설명 | 단계 |
| --- | --- | --- |
| **의도 분류** | 사용자 입력을 매물 추천 / 법률 상담 / 시세·안전 분석 / 보증 계산 4가지 의도로 분류. 의도에 따라 왼쪽 패널 자동 전환. 상단 배지로 현재 모드 표시. | MVP |
| **매물 추천** | 자연어에서 지역·평수·예산·유형 파싱. 매물 검색 후 최대 5개 카드 반환. 챗봇 카드 클릭 시 지도 해당 마커 포커스. 안전 점수 및 실거래가 대비 가격 평가 포함. | MVP |

> 챗봇 기반 / 로그인 필요

## F-4. 시세·안전 분석

> 지도·매물 상세 기반 조회는 비로그인 가능 / AI 에이전트 자연어 분석 요청은 로그인 필요

| 기능 | 상세 설명 | 단계 |
| --- | --- | --- |
| **시세 분석** | 매물 또는 지역 선택 시 최근 실거래가 추이, 전고점 대비 현황, 동일 평형 비교 등 상세 분석 제공. 지도와 매물 상세에 포함되는 공개 기능입니다. | MVP |
| **안전 분석** | 매물 주소 기준 반경 내 비상벨·보안등·파출소 밀도 점수화. 지도 오버레이와 연동해 시각적으로 표시합니다. 기본 조회는 공개 기능입니다. | MVP |
| **뉴스 RAG** | 시세 분석 응답에 관련 부동산 정책·시장 동향 뉴스 함께 제공. 왼쪽 패널에 뉴스 카드 표시. 최신성 유지를 위한 주기적 재인덱싱 필요. | 1.5차 |

## F-6. 인증

| 기능 | 상세 설명 | 단계 |
| --- | --- | --- |
| **회원가입** | 이메일 + 비밀번호. 이메일 중복 확인. 비밀번호 최소 8자, 영문+숫자 조합. 가입 완료 시 자동 로그인. | MVP |
| **로그인 / 로그아웃** | 이메일/비밀번호 로그인. JWT 액세스 토큰 + 리프레시 토큰 발급. 로그아웃 시 세션 종료 + 클라이언트 토큰 삭제. | MVP |
| **비로그인 접근** | F-1은 비로그인도 가능. F-2~F-5는 로그인 필요. 미로그인 챗봇 진입 시 로그인 유도 모달 표시. | MVP |
| **소셜 로그인** | 네이버 OAuth2.0. 최초 소셜 로그인 시 자동 회원가입. 이미 가입된 이메일이면 계정 연동. | 1.5차 |

## F-7. 찜하기

> 로그인 필요

| 기능 | 상세 설명 | 단계 |
| --- | --- | --- |
| **찜하기** | 매물 마커 팝업 또는 챗봇 추천 카드에서 찜 버튼으로 관심 매물 저장. 찜 목록 페이지에서 저장한 매물 조회. 찜한 매물은 지도에서 별도 마커 색상으로 표시. | MVP |

## F-8. 대화 세션 및 요약 메모리

> 로그인 필요

| 기능 | 상세 설명 | 단계 |
| --- | --- | --- |
| **대화 세션 관리** | 단일 세션에서 매물 추천·법률 상담·시세 분석을 끊김 없이 이어갈 수 있음. 세션은 의도·요약·최근 메시지·선택 매물 ID 등 상태를 유지. | 1.5차 |
| **요약 메모리** | 메시지 20개 초과 또는 누적 8,000자 초과 시 이전 대화 요약 + 최근 10개 메시지 + 현재 질문 조합으로 프롬프트 구성. 지역·예산·거래유형·선택 매물·법률 상담 맥락 보존. | 1.5차 |
| **대화 기록 조회** | 로그인 사용자는 이전 세션 목록과 대화 원문 재조회 가능. | 1.5차 |

## MVP 제외 사항

| 기능 | 제외 이유 |
| --- | --- |
| 실제 매물 중개 | 더미 매물로 대체. 중개사 매물 등록, 문의, 거래 상태 관리는 후순위. |
| 커뮤니티 | 게시글, 신고, 권한 처리 범위가 큼. |
| 뉴스 RAG | 크롤링 주기 관리, 재인덱싱 필요. 1.5차에서 추가. |
| HUG 정밀 판정 | 공시가격 API 연동 + 사용자 입력 복잡도. 1.5차에서 추가. |
| 등기부등본 AI 분석 | Claude Vision API 연동 필요. 확장 단계에서 추가. |
| 소셜 로그인 | OAuth 연동 복잡도. 1.5차에서 추가. |
| 대화 세션·메모리 | LangGraph 상태 관리 복잡도. 1.5차에서 추가. |
