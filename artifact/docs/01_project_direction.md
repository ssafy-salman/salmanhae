# 01. 서비스 개발 방향

## 서비스 정의

**살만해**는 AI 에이전트 기반 부동산 탐색 서비스입니다.

사용자가 자연어로 조건을 입력하면, AI 에이전트가 매물·시세·안전정보·계약 법률 정보를 연결해 추천과 설명을 제공합니다. 전월세·매매를 모두 다루며, 중개 서비스가 아닌 **계약 전 안전성 검증 중심** 서비스입니다.

## 해결하려는 문제

- 청년 1인 가구는 전세사기, 깡통전세, 허위매물 위험에 취약합니다.
- 등기부등본, 선순위채권, HUG 보증 기준 같은 계약 안전 정보를 스스로 해석하기 어렵습니다.
- 실거래가, 안전시설, 법률 정보가 여러 곳에 흩어져 있어 한 번에 확인하기 어렵습니다.
- 기존 부동산 서비스는 매물 탐색에 집중하며 "이 계약이 안전한가"에 대한 답을 주지 않습니다.

## 핵심 방향

살만해의 메인은 **AI 에이전트와 지도의 연동**입니다.

사용자는 지도에서 매물을 탐색하고, AI 에이전트에 자연어로 질문하면 에이전트가 지도·매물·시세·법률 정보를 연결해 답변합니다. 에이전트의 추천 결과는 지도 마커로 바로 표시됩니다.

```
지도에서 매물 탐색 (비로그인)
→ AI 에이전트에 자연어로 질문 (로그인)
    ├─ 매물 추천 → 지도 마커 연동
    ├─ 시세·안전 분석
    └─ 법률 상담 (임대차보호법, 전세사기특별법)
→ HUG 보증 간이 자가진단
→ "이 집 계약해도 되나?" 판단
```

## 서비스 포지셔닝

| 구분 | 기존 서비스 | 살만해 |
| --- | --- | --- |
| 네이버부동산, 직방 | 매물 탐색·중개 | — |
| 호갱노노 | 실거래가·시세 | — |
| **살만해** | — | AI 에이전트 기반 부동산 탐색 + 계약 안전성 검증 |

## 기술 스택

| 구분 | 기술 |
| --- | --- |
| Frontend | Vue 3, 네이버지도 SDK, Pinia, Axios |
| Backend | Spring Boot, Supabase Auth, PostgreSQL + pgvector |
| AI | Python FastAPI, LangGraph, Claude API |
| Infra | Cloud Run (Spring Boot + FastAPI 각각 배포) |
| Batch | Spring Scheduler 기반 공공데이터 수집 |
| External API | 국토교통부 실거래가 8종, 생활안전지도 API 5종, 딥서치 API |
